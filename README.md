# HuriosRally Frontend - Base de trabajo colaborativo

Este repositorio contiene la estructura base del frontend para trabajo en equipo.

## Que incluye esta base

- Archivos de configuracion esenciales (`package.json`, `vite.config.ts`, `tsconfig*`, `eslint.config.js`, etc.).
- Estructura de carpetas principal del proyecto.
- Carpetas internas vacias con `.gitkeep` para que Git las versione.

## Distribucion de trabajo

La idea es que cada integrante suba archivos de la carpeta que le corresponde (por ejemplo: `src/api`, `src/components`, `src/hooks`, etc.).

## Importante

- Los `.gitkeep` solo sirven para mantener carpetas vacias en el repositorio.
- Cuando agregues archivos reales en una carpeta, puedes eliminar su `.gitkeep`.
- Trabajar sobre la rama `dev` y abrir PR hacia `dev` segun el flujo del equipo.
